## 📖 Overview

**NeuralSentinel** is a cross-platform desktop application for auditing Machine Learning models entirely on-device. It combines an **Electron** frontend with a **Python/Flask** backend to deliver comprehensive evaluations across three audit dimensions:

- **Security** – robustness against adversarial attacks.
- **Privacy** – detection of information leakage and inference attack risks.
- **Fairness** – measurement of bias and prediction fairness.

All processing runs locally on the user's machine. No data is ever sent to external servers.

## 🛠️ Installation

### Option A — Quick Install (scripts)

**Windows:**
```bat
install.bat
```

**Linux:**
```bash
chmod +x install.sh && ./install.sh
```

The scripts create the Python virtual environment, install all Python and Node.js dependencies, and prepare the required data directory structure.

### Option B — Manual Install

#### 1. Install Node.js dependencies (Electron)

```bash
npm install
```

#### 2. Create Python virtual environment and install dependencies

```bash
cd backend
python -m venv venv

# Windows
venv\Scripts\activate

# Linux / macOS
source venv/bin/activate

pip install -r requirements.txt
```

> ⚠️ Installing TensorFlow, PyTorch and Foolbox may take several minutes. A stable internet connection is recommended.

## 🚀 Running the Application

### Development Mode

Requires two terminals:

**Terminal 1 — Python backend (Flask):**
```bash
cd backend
# (activate venv if needed)
python app.py
```
The backend starts at `http://127.0.0.1:5000`.

**Terminal 2 — Electron frontend:**
```bash
npm run dev
```

The application window opens automatically.

### Production Mode

```bash
# Windows
start.bat

# Linux
chmod +x start.sh && ./start.sh
```

In production mode, Electron automatically launches the compiled backend executable (`backend/app.exe` on Windows) and waits 2 seconds before showing the window.

### Models — `/api/models`

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/models` | List all registered models |
| `GET` | `/api/models/<model_id>` | Get a specific model |
| `POST` | `/api/models/upload` | Upload a new ML model |
| `GET` | `/api/models/<model_id>/architecture` | Get model architecture details |
| `DELETE` | `/api/models/<model_id>` | Delete a model |

**Supported model formats:** `.h5`, `.keras` (TensorFlow/Keras), `.pt`, `.pth` (PyTorch)

**Upload — `POST /api/models/upload`** (multipart/form-data):
```
file        — Model file (required)
name        — Descriptive name (required)
framework   — 'tensorflow' | 'pytorch' (default: 'tensorflow')
description — Optional description
```

### Datasets — `/api/datasets`

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/datasets/` | List all datasets |
| `GET` | `/api/datasets/<dataset_id>` | Get dataset info |
| `POST` | `/api/datasets/upload` | Upload a new dataset |
| `GET` | `/api/datasets/<dataset_id>/preview` | Preview images or tabular data |
| `DELETE` | `/api/datasets/<dataset_id>` | Delete a dataset |

**Supported formats:** `.npy`, `.npz`

**Upload — `POST /api/datasets/upload`** (multipart/form-data):
```
data_file   — NumPy data file (required)
labels_file — NumPy labels file (optional)
name        — Dataset name
description — Optional description
```

> ℹ️ If both data and labels are provided, their lengths are validated to match.

**Preview:** Automatically detects whether the data is images (3D/4D arrays) or tabular and returns up to 20 samples as base64-encoded PNGs (images) or a list (tabular).

| `POST` | `/api/evaluations/<eval_id>/start` | Start background execution |
| `GET` | `/api/evaluations/<eval_id>/status` | Poll status and progress |
| `GET` | `/api/evaluations/<eval_id>/logs` | Stream real-time logs |

| `GET` | `/api/evaluations/<eval_id>/images/<filename>` | Serve generated images |

**Create evaluation — `POST /api/evaluations/`**:
```json
{
  "model_id": "model-uuid",
  "dataset_id": "dataset-uuid",
  "metrics": ["Metric1", "Metric2"]
}
```

**Evaluation lifecycle:**
```
pending → running → completed
                 → cancelled
                 → error
```

**Completed evaluation response:**
```json
{
  "id": "uuid",
  "status": "completed",
  "progress": 100,
  "results": {
    "NeuralStrength": {
      "status": "completed",
      "score": 72.5,
      "category": "security",
      "details": {},
      "warnings": [],
      "recommendations": [],
      "visualization": "/api/evaluations/uuid/images/Metric1.png"

```

> ℹ️ Evaluations run in a background thread. Plugin stdout/stderr is captured and available via `/logs?since=<index>`.

## 🔌 Plugin System

Plugins are the core of the evaluation engine. The `PluginManager` recursively scans `backend/plugins/` at startup and auto-registers all valid plugins.

### Base Class `MetricPlugin`

All plugins must inherit from `MetricPlugin` (`backend/plugins/base.py`) and implement the required methods:

```python
from plugins.base import MetricPlugin

class MyMetric(MetricPlugin):

    def manifest(self) -> dict:
        """Plugin metadata (required)."""
        return {
            'name': 'My Metric',           # Unique plugin name
            'type': 'security',            # 'security' | 'privacy' | 'fairness'
            'version': '1.0.0',
            'description': 'What this metric measures',
            'parameters': {},              # Configurable parameters
            'author': 'Your Name',         # Optional
            'requirements': ['numpy']      # Optional extra dependencies
        }

    def build(self, model, config: dict):
        """Prepare the plugin with the model and configuration."""
        self.model = model

    def call(self, dataset) -> dict:
        """
        Compute the metric on the dataset.
        
        You may also implement __call__ instead of call — both are supported.
        The method may accept (dataset,) or (dataset, labels)
        depending on the __call__ signature.
        """
        return {
            'score': 0.85,            # float in [0, 1] — auto-normalized to [0, 100]
            'details': {},            # Additional information
            'warnings': [],           # Alerts
            'recommendations': []     # Improvement suggestions
        }

    def view(self):
        """Optional: return a matplotlib Figure for visualization."""
        import matplotlib.pyplot as plt
        fig, ax = plt.subplots()
        ax.plot(self.scores)
        return fig
```

### Plugin Deployment

| Type | Location | Description |
|---|---|---|
| Single file | `backend/plugins/custom/my_plugin.py` | Standalone Python file |
| Library | `backend/plugins/my_library/` | Folder with multiple modules |
| Upload via UI | Upload `.py` or `.zip` from the interface | Extracted and reloaded automatically |

> When uploading a `.zip` file, the zip filename (without extension) becomes the library directory name.

## 🧪 Basic Usage

### Step 1 — Load a Model

1. Navigate to the **Models** section.
2. Click **Upload Model**.
3. Select a `.h5` / `.keras` file (TensorFlow) or `.pt` / `.pth` (PyTorch).
4. Assign a descriptive name and choose the framework.

### Step 2 — Load a Dataset

1. Navigate to the **Datasets** section.
2. Click **Upload Dataset**.
3. Select the data file (`.npy` or `.npz`).
4. Optionally add a labels file (`.npy`).

> The dataset must contain test-set data in NumPy format. If the data is images, the system automatically detects this and shows a preview of the first 20 samples.

### Step 3 — Create an Evaluation

1. Navigate to the **Evaluation** section.
2. Select the model and dataset you previously uploaded.
3. Select the metrics (plugins) to run.
4. Click **Start Evaluation**.

The evaluation runs in the background. You can monitor real-time progress and logs from the same screen.

### Step 4 — View Results

1. Navigate to the **Results** section or check the evaluation history.
2. Review the score (0–100), details, warnings and recommendations for each metric.
3. Inspect plugin-generated charts (if available).

## 🔒 Security & Privacy

- ✅ **100% Local**: all data (models, datasets, results) stays on your machine.
- ✅ **No Telemetry**: no data is sent to external servers.
- ✅ **Context Isolation**: the Electron renderer runs with `nodeIntegration: false` and `contextIsolation: true`.
- ✅ **Local backend**: Flask listens only on `127.0.0.1:5000` (loopback interface).

## 🤝 Contributing

### Adding a New Plugin

1. Create a Python file in `backend/plugins/<your_library>/`.
2. Implement your class inheriting from `MetricPlugin`.
3. Define `manifest()`, `build()` and `call()` (or `__call__()`).
4. Restart the application or call `POST /api/plugins/reload`.

### Uploading a Plugin via the UI

1. Package your plugin as a `.py` file or as a `.zip` archive (zip name = library name).
2. Go to the **Plugins** section and use the upload button.
3. The system will extract, validate and load the plugin automatically.
